# 自动化科研Agent系统 — 三端分工计划

## 项目总览

```
agent_keyan/
├── backend/      ← Agent-后端 负责
├── desktop/      ← Agent-桌面端 负责
├── mobile/       ← Agent-手机端 负责
└── docs/         ← 共享文档
```

三端通过 REST API + WebSocket 通信，接口契约由 Agent-后端 定义，桌面端和手机端遵循。

---

## 通信契约（三端共识）

```
后端地址: http://{HOST}:8000
WebSocket: ws://{HOST}:8000/ws

核心API:
  POST   /api/tasks              创建研究任务
  GET    /api/tasks              任务列表
  GET    /api/tasks/{id}         任务详情
  GET    /api/tasks/{id}/status  任务状态(含当前模块进度)
  POST   /api/tasks/{id}/pause   暂停任务
  POST   /api/tasks/{id}/resume  恢复任务
  POST   /api/tasks/{id}/abort   终止任务
  GET    /api/tasks/{id}/logs    全程追溯日志
  GET    /api/tasks/{id}/output  产出物(论文PDF/代码/数据)
  POST   /api/tasks/{id}/review  人工审阅反馈(干预点)

WebSocket消息类型:
  { type: "progress",  task_id, module, step, percent, message }
  { type: "result",    task_id, module, data }
  { type: "need_review", task_id, module, content }  # 需要人工审阅
  { type: "error",     task_id, module, error }
  { type: "completed", task_id, output_url }
```

## Agent-后端

**负责范围**: `backend/` 目录

### 职责
1. FastAPI 服务搭建，定义所有 API 接口
2. 9模块研究流水线核心引擎
3. WebSocket 实时推送
4. 任务状态机与回退逻辑
5. 全程追溯日志系统
6. Docker 沙箱实验执行环境
7. 数据库(任务/日志/产出物持久化)

### 目录结构
```
backend/
├── main.py                         # FastAPI入口 + CORS配置
├── config.py                       # API keys, 模型配置, 路径配置
├── requirements.txt
│
├── api/
│   ├── routes.py                   # REST路由
│   ├── ws.py                       # WebSocket管理
│   └── schemas.py                  # Pydantic请求/响应模型
│
├── pipeline/
│   ├── orchestrator.py             # 流水线编排(M1→M9顺序执行)
│   ├── state.py                    # 状态机: pending→running→review→completed/failed
│   └── tracer.py                   # 追溯日志(每步输入/输出/耗时/token用量)
│
├── modules/
│   ├── base.py                     # 模块基类(统一接口)
│   ├── m1_literature.py            # 文献调研
│   ├── m2_gap_analysis.py          # 研究空白识别
│   ├── m3_idea_scoring.py          # Idea生成与打分
│   ├── m4_code_gen.py              # 代码仓库生成
│   ├── m5_experiment_design.py     # 实验设计
│   ├── m6_agent_runner.py          # Agent自动实验执行
│   ├── m7_analysis.py              # 结果分析 + 回退判断
│   ├── m8_paper_writing.py         # 论文写作
│   └── m9_review.py                # ICLR式评审打分
│
├── db/
│   ├── database.py                 # SQLite/PostgreSQL连接
│   └── models.py                   # ORM模型
│
└── sandbox/
    ├── Dockerfile                  # 实验沙箱镜像
    └── runner.py                   # 沙箱内代码执行器
```

### 开发阶段
| 阶段 | 任务 | 产出 |
|------|------|------|
| B1 | FastAPI骨架 + 任务CRUD API + SQLite | 可通过curl创建/查询任务 |
| B2 | 模块基类 + M1文献调研(调LLM API) | 输入主题→返回文献综述 |
| B3 | M2选题 + M3 Idea打分 | 输入主题→返回带评分的idea列表 |
| B4 | M4代码生成 + M5实验设计 | 根据idea生成实验代码 |
| B5 | M6 Agent实验执行(Docker沙箱) | 自动跑实验，产出结果 |
| B6 | M7结果分析 + 回退逻辑 | 判断达标/不达标，不达标回退M6 |
| B7 | M8论文写作 + M9评审打分 | 生成LaTeX论文 + 多维评分 |
| B8 | WebSocket实时推送 + 追溯日志 | 前端可实时接收进度 |
| B9 | 流水线编排完整串联 | CLI跑通M1→M9全流程 |

## 三端协作时序

```
阶段1: 后端先行（B1-B2）
  Agent-后端: 搭骨架 + 第一个模块可调用
  Agent-桌面端: 等待
  Agent-手机端: 等待

阶段2: 后端API就绪，前端启动（B3-B5 并行 D1-D3 并行 M1-M3）
  Agent-后端: 继续开发M2-M6模块
  Agent-桌面端: 初始化项目 + 对接API + Dashboard
  Agent-手机端: 初始化项目 + 对接API + 任务列表

阶段3: 三端联调（B6-B9 并行 D4-D8 并行 M4-M8）
  Agent-后端: 完成全流程 + WebSocket
  Agent-桌面端: 完成所有页面
  Agent-手机端: 完成所有页面

阶段4: 集成测试
  三端联合: 手机创建任务 → 电脑执行 → 手机/桌面实时查看 → 产出论文
```

## 依赖关系

```
Agent-后端 (无依赖，最先启动)
    │
    ├──► Agent-桌面端 (依赖后端API定义，阶段2启动)
    │
    └──► Agent-手机端 (依赖后端API定义，阶段2启动)
```

**关键约束**: 后端 API schemas (`api/schemas.py`) 定义后需同步给桌面端和手机端的 `types.ts`，保持三端类型一致。
