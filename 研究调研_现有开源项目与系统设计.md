# 自动化科研Agent系统 — 现有开源项目调研与系统设计

## 一、系统总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    前端 (frontend/)                          │
│         Web UI / 可视化面板 / 进度追踪 / 结果展示              │
└──────────────────────────┬──────────────────────────────────┘
                           │ REST API / WebSocket
┌──────────────────────────▼──────────────────────────────────┐
│                    后端 (backend/)                           │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ │
│  │ M1  │→│ M2  │→│ M3  │→│ M4  │→│ M5  │→│ M6  │→│ M7  │ │
│  │文献 │ │选题 │ │Idea │ │代码 │ │实验 │ │Agent│ │结果 │ │
│  │调研 │ │开题 │ │打分 │ │生成 │ │设计 │ │实验 │ │分析 │ │
│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └──┬──┘ │
│                                                  ↗   │     │
│                                        未达标回退 ┘   ▼     │
│                                              ┌─────┐       │

│                         │打分 │              └─────┘       │
│                         └─────┘                            │
└────────────────────────────────────────────────────────────┘
```

## 四、推荐技术路线（手机App + 电脑端）

### 整体架构：手机遥控 + 电脑执行

```
┌────────────────────┐         ┌─────────────────────────────────────┐
│   手机App (mobile/) │         │       电脑端 (desktop/)               │
│                    │ REST/WS │                                     │
│  ● 创建研究任务     │◄───────►│  FastAPI Server                     │
│  ● 实时进度监控     │         │    ├── WebSocket 推送               │
│  ● 中间结果审阅     │         │    ├── 研究流水线引擎                 │
│  ● 人工干预/审批    │         │    │   ├── M1 文献调研                │
│  ● 论文预览/下载    │         │    │   ├── M2 选题开题                │
│  ● 全程日志追溯     │         │    │   ├── M3 Idea打分               │
│                    │         │    │   ├── M4 代码生成                │
│  React Native      │         │    │   ├── M5 实验设计                │
│  或 Flutter        │         │    │   ├── M6 Agent实验执行            │
│                    │         │    │   ├── M7 结果分析(可回退M6)       │
└────────────────────┘         │    │   ├── M8 论文写作                │
                               │    │   └── M9 评审打分                │
                               │    ├── 沙箱执行环境(Docker)           │
                               │    └── 追溯日志系统                   │
                               └─────────────────────────────────────┘
```

### 技术路线评估与选择

#### 路线A: AutoResearchClaw 为基座 (推荐)
```
优势：
  - 23阶段最完整流水线，覆盖你要的全部9个模块
  - MIT协议，可自由修改
  - 零人工干预设计，天然适合远程手机控制
  - 4轮多Agent评审机制对应你的ICLR评审需求
  - 硬件感知沙箱，安全执行实验代码
  - 社区活跃(9.7K stars, 2026.3月仍在更新)
劣势：
  - 相对较新(v0.3.2)，可能有边角问题
  - 文献检索不如GPT-Researcher深度
改造计划：
  - 在其上层包装 FastAPI 服务
  - 替换文献模块为 GPT-Researcher + Semantic Scholar
  - 添加 WebSocket 进度推送供手机端订阅
```

#### 路线B: AI-Scientist v2 为基座
```
优势：Nature发表，学术公信力最强
劣势：模板依赖较重，代码生成弱，改造成本高
```

#### 路线C: 全自建拼接
```
优势：每个模块用最强工具
劣势：集成成本极高，接口对齐困难
```

**结论：选择路线A（AutoResearchClaw），文献模块用GPT-Researcher增强。**

## 五、具体技术栈

### 后端 (desktop/)
| 层 | 技术 | 说明 |
|---|------|------|
| API框架 | **FastAPI** + **WebSocket** | 高性能异步，手机端通信 |
| 研究引擎 | **AutoResearchClaw** | 23阶段核心流水线 |
| 文献增强 | **GPT-Researcher** + **Semantic Scholar API** | 替换/增强原有文献模块 |
| 文献问答 | **PaperQA2** | 超人级文献理解 |
| 实验管理 | **MLflow** | 实验追踪、对比、可视化 |
| 沙箱执行 | **Docker** | 安全执行生成的实验代码 |
| 数据库 | **SQLite** (轻量) / **PostgreSQL** (生产) | 任务状态、日志持久化 |
| LLM | **Claude API** / **OpenAI API** / 本地模型 | 可配置多后端 |

### 手机App (mobile/)
| 层 | 技术 | 说明 |
|---|------|------|
| 框架 | **React Native** (推荐) 或 Flutter | 跨平台iOS+Android |
| 状态管理 | Zustand / Redux | 任务状态同步 |
| 实时通信 | WebSocket客户端 | 接收进度推送 |
| 文件预览 | PDF/LaTeX渲染组件 | 论文预览 |

### 电脑桌面端 (desktop-ui/) — 后期
| 层 | 技术 | 说明 |
|---|------|------|
| 框架 | **Electron** + React 或 **Tauri** | 桌面GUI |
| 终端 | 内嵌xterm.js | 查看实验日志 |
