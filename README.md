## 项目简介

随着科研产出的快速增长，科研人员面临海量学术文献数据的挑战。**ScholarMind** 是一个基于AI大模型的智能文献分析系统，能够自动从海量学术文献中提取关键信息，进行深度分析、知识挖掘和交互式编辑，为科研人员提供全面的文献分析报告和洞察。

## 功能模块

### 1. 📝 文献数据收集与预处理

- 支持从 **arXiv** 和 **Semantic Scholar** 两大学术数据库自动检索
- 自动数据清洗：去重、文本标准化、日期格式统一
- 灵活的过滤功能：按年份、关键词筛选文献
- **引用关系探索**：查看任意论文的引用文献和参考文献，追踪学术脉络
- 本地 JSON 持久化存储

### 2. 🔬 关键信息提取与知识图谱

- **双模式关键词提取**：
  - LLM 模式：调用大模型提取关键词、研究方法、核心贡献、研究领域
  - TF-IDF 模式：无需 API 密钥，基于统计方法自动提取
- **三类知识图谱**：
  - 论文-作者-关键词知识图谱（三元组网络）
  - 作者合作网络（共同作者关系与合作强度）
  - 关键词共现网络（研究主题关联）
- **交互式图谱可视化**：基于 `streamlit-agraph` 的可拖拽、可探索网络图
- **网络分析指标**：度中心性、介数中心性、网络密度、连通分量

### 3. 📊 主题建模与趋势分析

- **两种主题建模方法**：
  - LDA（Latent Dirichlet Allocation）主题模型
  - TF-IDF + KMeans 文本聚类
- **可视化分析**：
  - 主题分布饼图 + 聚类散点图
  - 关键词频率柱状图 + 词云
  - 关键词年度热力图
- **年度趋势分析**：论文发表数量、引用量随时间变化
- **热点识别**：自动识别近年高频关键词和高引用论文

### 4. 💡 智能推荐与个性化服务

- **用户画像系统**：自动记录研究兴趣、阅读历史、收藏偏好、搜索记录
- **个性化推荐**：基于 TF-IDF + 余弦相似度的内容推荐，排除已读论文
- **相似论文发现**：选定论文后自动推荐语义相似文献
- **阅读管理**：论文标记已读/收藏，兴趣关键词自动更新

### 5. 🤖 AI 文献助手（LLM 驱动）

- **论文问答**：基于已检索文献内容向 AI 提问，获取带引用的回答
- **文献综述生成**：一键生成包含领域概述、研究方向、关键发现、未来趋势的综述报告
- **结构化信息提取**：自定义字段（如"研究方法"、"数据集"、"关键发现"），AI 从每篇论文中提取并生成对比表格
- 支持对话历史，可进行多轮追问

### 6. 📋 文献管理与导出

- **多格式导出**：CSV、JSON、**BibTeX**（可直接导入 LaTeX）
- **文献标注系统**：为论文添加/编辑/删除个人笔记
- **数据统计面板**：总论文数、来源分布、独立作者数、总引用量

## 快速启动

### 环境要求

- Python 3.10+
- pip

### 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/yibol9768-alt/ScholarMind---.git
cd ScholarMind---

# 2. 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置 LLM API（可选）
cp .env.example .env
# 编辑 .env 文件，填入你的 API Key
```

### 启动应用

```bash
streamlit run app.py
```

浏览器访问 `http://localhost:8501` 即可使用。

### LLM 配置说明

系统支持任何 OpenAI 兼容的 API（OpenAI、DeepSeek、通义千问、Ollama 等）：

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
MODEL_NAME=gpt-4o-mini
```

**常见配置示例：**

| 服务商 | BASE_URL | MODEL_NAME |
|--------|----------|------------|
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| Ollama (本地) | `http://localhost:11434/v1` | `llama3` |

> **无需 LLM 也可运行**：未配置 API Key 时，系统自动使用 TF-IDF 统计方法替代大模型进行关键词提取和分析。LLM 相关功能（论文问答、综述生成、结构化提取）需要配置 API。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端框架** | Streamlit | 交互式 Web 界面 |
| **数据可视化** | Plotly, streamlit-agraph, streamlit-echarts | 图表 + 交互式图谱 + 词云 |
| **数据源** | arXiv API, Semantic Scholar API | 学术文献检索与引用追踪 |
| **NLP/ML** | scikit-learn | TF-IDF, LDA, KMeans, 余弦相似度 |
| **LLM** | OpenAI 兼容 API | 问答、综述、结构化提取 |
| **知识图谱** | NetworkX | 网络构建与分析 |
| **数据处理** | Pandas, NumPy | 数据清洗与标准化 |

## 项目结构

```
ScholarMind/
├── app.py                     # Streamlit 主应用入口 (6个功能标签页)
├── config.py                  # 全局配置（API Key、路径等）
├── requirements.txt           # Python 依赖
├── .env.example               # 环境变量模板
├── data/                      # 本地数据存储
│   ├── papers.json            # 文献数据缓存
│   ├── user_profile.json      # 用户画像
│   └── notes.json             # 文献标注笔记
├── modules/
│   ├── data_collector.py      # 文献数据收集与预处理（arXiv + S2 + 引用追踪）
│   ├── info_extractor.py      # 关键信息提取与知识图谱构建
│   ├── topic_analyzer.py      # 主题建模与趋势分析
│   ├── recommender.py         # 智能推荐引擎（用户画像 + 相似度计算）
│   └── visualizer.py          # 可视化图表生成（Plotly + ECharts）
└── static/                    # 静态资源
```
